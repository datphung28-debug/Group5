# Database-Backed Audit Log Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist selected high-value user actions in MongoDB and replace the dormant activity-history screen with admin-only real audit data.

**Architecture:** Keep the existing `route -> controller -> Mongoose model` structure. Controllers call a best-effort `createAuditLog()` helper only after their primary database operation succeeds; an admin-only read controller exposes filtered audit rows adapted for the existing React page.

**Tech Stack:** Express, Mongoose, Node test runner, React, Vite, Axios, Ant Design

---

- [ ] **Step 3: Implement schema and helper**

Create `AuditLog` with the spec fields, enum values and indexes. Implement:

```js
export const createAuditLog = async ({
  req,
  user,
  action,
  module,
  target,
  description,
  status = "success",
}) => {
  try {
    const actor = user || req?.user;
    const forwardedFor = req?.headers?.["x-forwarded-for"];
    return await AuditLog.create({
      user: actor?._id,
      userName: actor?.name,
      userRole: actor?.role,
      action,
      module,
      target,
      description,
      status,
      ipAddress: forwardedFor?.split(",")[0].trim() || req?.ip || req?.socket?.remoteAddress || "",
      userAgent: req?.get?.("user-agent") || req?.headers?.["user-agent"] || "",
    });
  } catch (error) {
    console.error("Không thể ghi audit log:", error);
    return null;
  }
};
```
